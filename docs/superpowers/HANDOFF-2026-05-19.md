## Known-good workflow patterns (from this session)

When you start a new batch:

1. **Brainstorm with user** (skill: `superpowers:brainstorming`). Lock the contract decisions via Q&A before writing anything.
2. **Dispatch 3 adversarial reviewers** against the brainstorm-locked design. Pre-load each with a specific skeptical hypothesis to falsify. Don't ask "any concerns?" — that returns noise.
   - R1: silent-failure-hunter (hunt for behavior-drift bugs)
   - R2: code-reviewer (hunt for contract-composition bugs)
   - R3: pr-test-analyzer (hunt for tautology + test-coverage gaps)
3. **Apply all real findings inline** to the spec before plan-writing. The spec self-justifies via the inline R1/R2/R3 references.
4. **Write the plan** (skill: `superpowers:writing-plans`). Self-review for placeholders + type-consistency. Commit.
5. **Subagent-driven execution** (skill: `superpowers:subagent-driven-development`). Per-task implementer + spec-compliance review + code-quality review. Commit-tasks executed by controller, not subagent.
6. **Post-batch holistic code review** (skill: `superpowers:requesting-code-review`). Different things found vs. per-task reviews.

## Things to NOT do reflexively

- **Don't push without checking ahead-count** — parallel sessions push too. `git status -sb` first.
- **Don't `git add -A`** — pre-existing modified files in the tree are not yours. Stage specific files only.
- **Don't trust per-task implementer claims of "all tests pass"** — they may only run the test file they touched. Run `bun run test` full suite at the controller-level before each commit.
- **Don't claim "no equivalent C13 test cleanup needed"** in a migration spec without running `grep -rn '<wire_type>' src/ipc.test.ts` first. The Batch 2G spec missed 5 legacy describe blocks because I didn't grep; the subagent caught 4 of them and the code reviewer caught the 5th.
- **Don't add lint-error-prone helpers ahead of their consumer** — eslint's `no-unused-vars` will block the commit. Move helpers WITH their first user, not before.
