### Commit 2 — `88688ee3` harden (round-2 review follow-up)

Closes two related silent-failure-wedge variants that the first round's L1 fix didn't cover. Both were verified at source via independent runtime tests BEFORE production edits.

- **H1** — outer `Array.isArray` guard validated the wrapper but not the element shape. `{results: [null]}` / `{results: [{}]}` bypassed L1 and threw `TypeError` inside `.map()` body (`r.title` / `r.score.toFixed()`), caught by the outer try/catch as the SAME misleading `'QMD unavailable: ...'` message L1 was supposed to close.
- **H1.6** — when `rawText` was literal `"null"`, `JSON.parse` produced the JS primitive `null`, and the L1 line `Array.isArray(parsed.results)` threw on the null dereference BEFORE the L1 guard could fire. Same wedge, one level shallower.

The harden commit added two new guards in sequence between `JSON.parse` and the `.map()`:

1. **parsed-shape guard** (H1.6): rejects `parsed === null`, primitives (number, boolean, string), and arrays. Distinct `'QMD returned malformed results array'` message + `logger.warn` with `parsedType`. Type annotation changed from `as { results?: ... }` (which lied to TS about runtime shape) to `as unknown`, with a narrowed cast to `parsedObj` after the guard.
2. **element-shape predicate** (H1): strict-reject — any malformed element rejects the whole response (vs lenient filter, which would mask QMD schema drift per `feedback_silent_failure_wedge.md`). Predicate checks `file`/`title`/`snippet` are strings AND `score` is a finite number (`Number.isFinite` blocks NaN, Infinity).

Plus a fix to flaky test 22 (crystallize_skill path-traversal): the old assertion `fs.existsSync(path.join(agentsTmpRoot, '..', 'etc'))` was host-dependent (macOS sometimes creates an /etc subdir in $TMPDIR) and stale state from prior test runs caused a false-positive failure. Replaced with environment-robust checks on the specific would-be-traversal target plus an `agentsTmpRoot` emptiness assertion.
