# npm to Bun Migration — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate NanoClaw's host-side runtime from Node.js/npm to Bun for faster startup, better runtime performance, and faster dependency management.

**Architecture:** Phased migration — package manager first, then runtime + SQLite together, then cleanup. Container stays on Node 22. Vitest kept as test runner.

**Tech Stack:** Bun (runtime + package manager), bun:sqlite, TypeScript, Vitest, pino

**Spec:** `docs/superpowers/specs/2026-03-28-npm-to-bun-migration-design.md`

export const logger = pino(
  { level: process.env.LOG_LEVEL || 'info' },
  PinoPretty({ colorize: true }),
);

// Route uncaught errors through pino so they get timestamps in stderr

  logger.fatal({ err }, 'Uncaught exception');
